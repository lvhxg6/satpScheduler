# Review Report

## 摘要
本次比对了整体PRD与后端PRD，核心调度/盲时/手动触发/数据模型等大部分逻辑一致。发现1处高优先级逻辑冲突（重试策略），以及1处低优先级的需求覆盖差异（轮次列表字段与筛选项未在后端接口明确）。文档篇幅较长，已逐段对核心规则比对，但仍可能存在未穷尽处。

## 问题清单

### 高
1) **重试策略存在冲突（整体PRD vs 后端PRD，且后端内部自相矛盾）**
   - 整体PRD明确“调度异常或AMR接口调用失败时不进行自动重试”（`docs/任务自动下发配置-PRD-20260129120000.md:1101-1104`），并在AMR异常处理中再次强调“不进行自动重试”（`docs/任务自动下发配置-PRD-20260129120000.md:1057-1065`）。
   - 后端PRD在可靠性要求中写“调度任务异常时记录日志并延迟重试”（`docs/任务自动下发配置-PRD-后端-20260129120000.md:952-955`），但同一文档在AMR异常处理又写“不进行自动重试”（`docs/任务自动下发配置-PRD-后端-20260129120000.md:872-880`）。
   - 这将直接影响是否触发自动重试与任务幂等/重复创建策略，属于核心逻辑冲突。

### 低
1) **轮次管理的筛选/展示字段需求在后端接口中未明确**
   - 整体PRD对“自动扫描情况跟踪”列出筛选字段、列表字段与执行时间展示规则（`docs/任务自动下发配置-PRD-20260129120000.md:610-665`），但后端PRD仅描述轮次列表接口“支持分页、筛选”，未明确需要支持的筛选项与返回字段（`docs/任务自动下发配置-PRD-后端-20260129120000.md:926-931`）。
   - 若后端接口参数/返回字段不对齐前端展示需求，可能导致联调阶段补需求或返工。

## 修改建议
1) 统一重试策略并在两份PRD中保持一致：
   - 若以“**不自动重试**”为准，删除后端PRD 12.2 中“延迟重试”描述，并在可靠性要求中与AMR异常处理一致。
   - 若以“**允许延迟重试**”为准，需补充：重试触发条件、重试次数/间隔、幂等与重复任务处理、重试失败后落盘与通知策略。
2) 在后端PRD的轮次管理接口中补齐筛选条件与返回字段清单（至少覆盖整体PRD 8.3.2/8.3.3 所需字段），或在整体PRD中明确这些字段由前端计算/隐藏，避免接口期望不一致。

## 一致性校验
- 除上述问题外，调度机制、盲时处理、手动触发流程、数据模型与边界场景的描述整体一致。

## 风险/待确认点
- 需业务/架构确认“调度/AMR失败是否允许自动重试”的最终策略，并同步到双方PRD。
- 轮次列表筛选/字段需求是否必须由后端接口提供，需产品与后端确认，否则存在联调返工风险。
