# Review Report

## 摘要
本次为仅文档审查，聚焦“整体PRD中的前端逻辑是否完整拆分到前端PRD”。结论：前端PRD覆盖了主要页面与交互，但仍存在若干关键前端逻辑缺失或与整体PRD不一致的点，可能导致实现偏差或用户预期不一致。

## 问题清单
- **高**：关闭自动下发的提示文案与整体PRD规则冲突。前端PRD写明“关闭后将取消所有待执行的调度任务”，但整体PRD仅要求取消**自动**pending，不影响手动pending。见 `docs/任务自动下发配置-PRD-前端-20260129120000.md` §10.3 与 `docs/任务自动下发配置-PRD-20260129120000.md` §4.2.7。
- **中**：手动下发存在待执行手动pending时的更新/取消规则缺失。整体PRD要求“定时触发时更新原pending，立即执行时取消原pending并新建，且round_no/round_tag处理不同”，前端PRD仅给出校验规则，缺少该交互/提示与行为说明。见 `docs/任务自动下发配置-PRD-20260129120000.md` §8.5.1 与 `docs/任务自动下发配置-PRD-前端-20260129120000.md` §8.5。
- **中**：模板选择的“可为空”逻辑未体现。整体PRD明确“若工具无可选模板，则不需要选择，使用工具内置默认模板”，前端PRD未说明模板可为空，容易导致前端强制校验或UI阻塞。见 `docs/任务自动下发配置-PRD-20260129120000.md` §7.1/§7.2 与 `docs/任务自动下发配置-PRD-前端-20260129120000.md` §7.1/§7.3。
- **中**：“失败业务系统”语义不一致。整体PRD强调轮次状态反映**任务创建结果**，“失败业务系统”为**任务创建失败**；前端PRD表述为“扫描失败”，易让用户理解为执行失败。见 `docs/任务自动下发配置-PRD-20260129120000.md` §8.3.3/§8.3.4 与 `docs/任务自动下发配置-PRD-前端-20260129120000.md` §8.3.3。
- **低**：盲时判断优先级与“盲时仅控制下发时机、不影响执行时长”规则未在前端PRD体现，可能导致前端提示/帮助文案缺失或与后端判定不一致。见 `docs/任务自动下发配置-PRD-20260129120000.md` §5.4/§5.5 与 `docs/任务自动下发配置-PRD-前端-20260129120000.md` §5。
- **低**：外部漏洞扫描资产过滤规则未在前端PRD体现。整体PRD要求外部漏洞仅查询“应用”类型资产并在无资产时跳过业务系统，前端PRD未说明，可能影响用户预期与异常提示。见 `docs/任务自动下发配置-PRD-20260129120000.md` §6.2 与 `docs/任务自动下发配置-PRD-前端-20260129120000.md` §6。
- **低**：前端PRD中“本轮”定义重复出现两次，影响可读性与维护。见 `docs/任务自动下发配置-PRD-前端-20260129120000.md` §8.5.2。

## 修改建议
- 将“关闭自动下发”的提示文案与整体PRD对齐，明确仅取消**自动**pending，不影响手动pending，并在前端PRD补充对应交互说明。
- 在“手动下发新一轮次”补充整体PRD中的完整逻辑：存在手动pending时的更新/取消行为、round_no/round_tag变化、是否需要提示用户“将覆盖已有定时/将取消已有定时”等。
- 在“自动下发模式”补充“模板可为空”的约束与UI处理（无模板时禁用选择或显示默认模板提示，保存时不校验必选）。
- 将“失败业务系统”改为“任务创建失败的业务系统”，并在列表说明中强调轮次状态反映任务创建结果；如需展示执行失败另走AMR查询。
- 在盲时配置部分补充优先级与“仅控制下发、不影响执行”的规则，确保前端提示、帮助文案与后端判定一致。
- 在资产范围部分补充“外部漏洞仅应用资产”与“无资产则跳过业务系统”的用户可见说明或提示策略。
- 删除重复的“本轮”定义。

- “关闭自动下发”的提示与行为以整体PRD为准吗？如有调整，请同步两份文档。
