# Review Report

## 摘要
整体结构完整，但在“调度/并发/失败语义/执行顺序”上存在多处表述不一致或逻辑缺口，可能导致实现与预期偏差。建议先统一关键规则（并发、失败定义、创建方式）再细化边界场景。

## 问题清单
- 高：任务创建方式表述冲突，`6.2`要求“按顺序依次创建”，`4.2.9`要求“批量创建且本轮结束”，两者不可同时成立。`docs/任务自动下发配置-PRD-20260129120000.md:529`
- 高：关闭自动下发会取消“手动 pending”，与“手动/自动可共存”的设计目标冲突，可能误伤用户已排期的手动任务。`docs/任务自动下发配置-PRD-20260129120000.md:266`
- 高：失败状态未覆盖“无资产”但多处将“无资产”记为 failed，导致状态定义与实际写法不一致（包括表格、边界处理、失败提示）。`docs/任务自动下发配置-PRD-20260129120000.md:462`
- 中：并发控制后的“拒绝重试”机制不一致：生命周期里写10分钟重试，但轮次判断/手动流程图没有统一该机制，容易造成实现分歧。`docs/任务自动下发配置-PRD-20260129120000.md:188`
- 中：盲时规避算法缺少“循环重新校验”的明确说明，遇到连续盲时可能仍落入不可执行窗口。`docs/任务自动下发配置-PRD-20260129120000.md:120`
- 中：“本轮未扫描到的资产”语义不清，是否指“最近一轮自动调度”或“本月所有自动轮次合并”未定义，影响实现口径。`docs/任务自动下发配置-PRD-20260129120000.md:772`
- 中：手动触发的并发控制时机不一致：流程图在“点击”时拒绝，但核心调度逻辑在“触发时”判定，需统一。`docs/任务自动下发配置-PRD-20260129120000.md:700`
- 低：调度“等待逻辑”处同时出现“等待任务执行完成更新轮次状态”和“任务一次性创建即结束”描述，容易误解是否等待“任务执行完成”。`docs/任务自动下发配置-PRD-20260129120000.md:203`

## 修改建议
- 统一任务创建方式：若支持批量接口，去掉“按顺序依次创建”；若要求顺序，就改为逐个创建并说明性能与失败处理策略。
- 明确“关闭自动下发”的边界：仅关闭自动调度（不影响手动 pending）或全局停用（需在手动章节补充说明）。
- 统一“无资产”的状态与提示：要么补充 failed 定义包含“无资产”，要么引入 skipped/empty 语义并全局一致。
- 统一拒绝调度后的重试策略：明确是否仅自动 pending 重试，还是手动 pending 也重试，并在相关章节/流程图中一致化。
- 盲时规避加入“循环校验”说明：每次跳转后重新按优先级校验直到找到可执行时间。
- 明确“本轮未扫描到的资产”定义范围，并给出示例（例如：指最近一次自动轮次）。
- 统一手动触发并发控制时机：建议“创建 pending 不拒绝，到执行时判定”，或相反，必须全局一致。
